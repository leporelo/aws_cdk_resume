About this

Website is part of AWS cloudresumechallenge.dev challenge, where you build your web resume according to specific requirements.
Website challenge you to create a static HTML website hosted on S3, with CDN, DNS, and API retrieving visitors number from DB, automated with CICD.

I was supposed to use Githuib actions and SAM approach to automate website deployment, but I used CDK to do everything instead.

What is CDK?
With CDK I have a freedom in using my favourite language to write infrastructure code. CDK builds on CloudFormation and uses its engine to provission resources.
So I dont need to use declarative approach to write YAML (or JSON) to create EC2 but to actually use imperative way to code it. Yay!

For EC2 in CloudFormation I would normaly start with something like this: 
Resources:
  MyInstance:
  Type: AWS::EC2::Instance
  Properties:
    AvailabilityZone: us-east-1a
    ImageId: ami-009d6802948d06e52
    InstanceType: t2.micro

But in CDK i can write it as:
instance = ec2.Instance(self, 'Instance',
  instance_type='t2.micro',
  availability_zone='us-east-1a',
  image_id='ami-009d6802948d06e52')
